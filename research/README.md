# Pesquisa aplicada

## Objetivo

Organizar a camada de pesquisa do projeto, incluindo hipóteses, amostras, baselines e notas de revisão humana.

## Estrutura

```text
research/
├── hypotheses/
├── sampling/
├── baselines/
└── review-notes/
```

## Função desta área

Esta pasta não armazena dados operacionais do pipeline. Ela armazena a inteligência metodológica complementar usada para validar, testar e refinar o projeto.

## Regras

* hipóteses não são fatos;
* amostras devem ser justificadas;
* baselines devem ser versionados;
* revisões humanas devem registrar justificativa.
