# Prompt — review — TEMPLATE

## Como usar este modelo

Copie este arquivo para um prompt de revisão específico e mantenha apenas orientações compatíveis com o artefato revisado.

## Nome do prompt

Defina um nome curto e funcional.

## Objetivo

Descreva qual revisão será feita.

## Quando usar

Usar para revisar alertas, documentos metodológicos, grupos comparáveis, relatórios e hipóteses.

## Entradas disponíveis

- alerta ou documento alvo
- baseline relacionado
- grupo comparável
- notas anteriores
- governança e decision log

## Saída esperada

Registre se a saída será parecer curto, lista de correções, pontos incertos ou recomendação de status.

## Restrições obrigatórias

- separar fato, hipótese e inferência
- não reescrever tudo sem necessidade
- apontar riscos metodológicos concretos
- preservar linguagem neutra

## Critérios de validação

- inconsistências reais identificadas
- revisão objetiva
- recomendação clara
- incerteza registrada quando aplicável

## Prompt

Objetivo:
Descreva o objeto e o foco da revisão.

Entradas:
Liste os documentos e referências que podem ser usados.

Saída esperada:
Descreva o formato do parecer e o nível de concisão esperado.

Restrições:
1. Separe fatos, hipóteses e inferências.
2. Não trate alerta como prova.
3. Aponte apenas problemas reais e verificáveis.
4. Registre o que permanece incerto.
5. Sugira correções proporcionais ao problema encontrado.

Validação:
Explique como verificar se a revisão ficou objetiva, útil e aderente ao escopo.
