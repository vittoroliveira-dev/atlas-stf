# Checklist de auditoria — governança

## Objetivo

Verificar se as exigências mínimas de governança, rastreabilidade e auditoria foram cumpridas.

## Aplicação

Usar antes de considerar uma etapa do projeto formalmente consolidada.

## Itens obrigatórios

- [ ] As fontes usadas estão registradas.
- [ ] As decisões relevantes estão no decision log.
- [ ] Os riscos relevantes estão no risk register.
- [ ] Os campos principais estão no data dictionary.
- [ ] Os limites do recorte foram documentados.
- [ ] As incertezas foram marcadas.
- [ ] O artefato revisado está versionado.
- [ ] Existe responsável identificado.
- [ ] Existe data de atualização.
- [ ] Nenhum arquivo de código novo ou alterado ultrapassa 500 linhas sem justificativa explícita.
- [ ] O material pode ser auditado por terceiro sem contexto oral.

## Critérios de bloqueio

A etapa deve ser interrompida se:
- não houver rastreabilidade mínima;
- decisões relevantes não estiverem registradas;
- a documentação depender de conhecimento implícito.
