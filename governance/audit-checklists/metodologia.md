# Checklist de auditoria — metodologia

## Objetivo

Verificar se uma decisão metodológica ou documento metodológico do projeto está claro, defensável e consistente com o restante da estrutura.

## Aplicação

Usar antes de aprovar:
- definição de baseline;
- regra de comparabilidade;
- definição de score;
- mudança de escopo analítico;
- novo protocolo de revisão.

## Itens obrigatórios

- [ ] O problema metodológico está claramente descrito.
- [ ] A decisão distingue contexto, problema e escolha.
- [ ] Há alternativas consideradas.
- [ ] A justificativa está escrita de forma objetiva.
- [ ] Os impactos foram registrados.
- [ ] Os riscos foram registrados.
- [ ] Os artefatos afetados foram identificados.
- [ ] A decisão é compatível com as regras do agente.
- [ ] A decisão não extrapola o escopo atual sem registro formal.
- [ ] Há coerência com o decision log.

## Critérios de bloqueio

A etapa deve ser interrompida se:
- a decisão metodológica não puder ser explicada claramente;
- não houver justificativa;
- houver conflito não resolvido com documentação já aprovada.
