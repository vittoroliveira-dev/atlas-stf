# Checklist de auditoria — grupos comparáveis

## Objetivo

Validar se um grupo comparável foi definido de forma explícita e defensável.

## Aplicação

Usar este checklist antes de aceitar qualquer baseline ou alerta.

## Itens obrigatórios

- [ ] O grupo possui identificador.
- [ ] A versão da regra foi registrada.
- [ ] Os critérios de composição estão explícitos.
- [ ] A janela temporal está definida.
- [ ] O tamanho do grupo foi calculado.
- [ ] O grupo não é amplo demais para perder sentido.
- [ ] O grupo não é estreito demais para perder utilidade.
- [ ] Há justificativa para os campos escolhidos.
- [ ] O grupo pode ser explicado em linguagem simples.
- [ ] Há risco metodológico conhecido registrado.

## Critérios de bloqueio

A etapa deve ser interrompida se:
- o grupo não puder ser descrito claramente;
- o baseline não estiver explícito;
- o grupo tiver sido montado apenas por palavra solta ou por critério irrelevante.
