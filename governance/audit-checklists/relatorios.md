# Checklist de auditoria — relatórios

## Objetivo

Garantir que relatórios do projeto sejam consistentes, auditáveis e metodologicamente prudentes.

## Aplicação

Usar este checklist antes de aprovar qualquer relatório técnico ou executivo.

## Itens obrigatórios

- [ ] O relatório identifica o período analisado.
- [ ] O relatório identifica as fontes utilizadas.
- [ ] O relatório identifica limitações relevantes.
- [ ] O relatório separa fatos, hipóteses e inferências.
- [ ] O relatório não usa linguagem acusatória.
- [ ] O relatório informa os critérios de comparabilidade.
- [ ] O relatório informa os critérios dos alertas.
- [ ] O relatório contém observações metodológicas.
- [ ] O relatório registra riscos ou incertezas.
- [ ] O relatório aponta próximos passos compatíveis com o escopo.

## Critérios de bloqueio

O relatório não deve ser liberado se:
- omitir fontes;
- omitir limitações;
- tratar alerta como prova final;
- misturar interpretação forte com dado frágil.
