# Política de Segurança

## Escopo

Use esta política para reportar:

- vulnerabilidades técnicas no código ou na configuração do repositório;
- exposição indevida de dados, credenciais ou artefatos sensíveis;
- falhas que comprometam rastreabilidade, integridade ou uso responsável do projeto.

## Como reportar

Se o problema puder causar dano imediato, evite abrir issue pública com detalhes exploráveis. Abra uma comunicação privada com os mantenedores pelo canal disponível no repositório ou, na ausência de canal privado explícito, abra uma issue pública com descrição mínima do impacto e solicite tratativa reservada.

## O que incluir no reporte

- resumo do problema;
- impacto observado;
- artefatos afetados;
- passos para reproduzir, quando seguro;
- evidências disponíveis;
- mitigação temporária, se houver.

## O que não fazer

- não publique credenciais, dados sensíveis ou detalhes exploráveis sem necessidade;
- não trate limitação metodológica comum como vulnerabilidade sem explicar o risco concreto;
- não use o processo de segurança para contornar as regras metodológicas do projeto.
